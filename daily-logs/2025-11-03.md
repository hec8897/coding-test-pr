## 📝 풀이한 문제

### 1️⃣ 전화번호 목록 (Hash/Set) - 복습
**파일**: `모의고사/1차_모의고사/1_전화번호_목록/solution.ts`

#### 핵심 개념
- **Hash (Set)**: 효율적인 중복 검사
- **부분 문자열**: `substring()`, `startsWith()` 활용
- **시간 복잡도**: O(n × m) (n: 전화번호 개수, m: 평균 길이)

#### 주요 로직
```typescript
// Set으로 빠른 조회
const phoneSet = new Set(phone_book);

// 각 전화번호의 모든 접두사 확인
for (let i = 1; i < phone.length; i++) {
  if (phoneSet.has(phone.substring(0, i))) {
    return false;
  }
}
```

#### 배운 점
- Set의 `has()` 메서드는 O(1)로 매우 빠름
- 부분 문자열 확인은 `substring()`으로 효율적
- 이미 풀었던 문제라 빠르게 해결 가능

### 2️⃣ 짝지어 제거하기 (Stack) - 복습
**파일**: `모의고사/1차_모의고사/2_짝지어_제거하기/solution.ts`

#### 핵심 개념
- **Stack (LIFO)**: 마지막에 들어온 것이 먼저 나감
- **연속 문자 제거**: Stack의 top과 현재 문자 비교
- **시간 복잡도**: O(n)

#### 주요 로직
```typescript
const stack: string[] = [];

for (const char of s) {
  // Stack이 비어있거나 top과 다르면 push
  if (stack.length === 0 || stack[stack.length - 1] !== char) {
    stack.push(char);
  } else {
    // top과 같으면 pop (짝지어 제거)
    stack.pop();
  }
}

return stack.length === 0 ? 1 : 0;
```

#### 배운 점
- Stack의 top 확인: `stack[stack.length - 1]`
- 연속된 같은 문자 제거는 Stack의 전형적인 활용
- else 분기에서 push를 하지 않는 것이 핵심 (이전에 실수했던 부분)

### 3️⃣ 올바른 괄호 (Stack) - 복습
**파일**: `모의고사/1차_모의고사/3_올바른_괄호/solution.ts`

#### 핵심 개념
- **Stack (짝맞추기)**: 여는 괄호 push, 닫는 괄호 pop
- **조기 반환**: Stack이 비어있는데 닫는 괄호가 오면 false
- **시간 복잡도**: O(n)

#### 주요 로직
```typescript
const stack: string[] = [];

for (const char of s) {
  if (char === '(') {
    stack.push(char);
  } else {
    // 닫는 괄호인데 Stack이 비어있으면 false
    if (stack.length === 0) return false;
    stack.pop();
  }
}

// 모든 괄호가 짝지어졌으면 Stack이 비어야 함
return stack.length === 0;
```

#### 배운 점
- 조기 반환 조건을 확인하는 것이 중요
- Stack이 비어있는지 확인 후 pop 해야 안전
- 괄호 문제는 Stack의 가장 기본적인 활용

  } else {

### 3. Stack 패턴 (짝맞추기)
```typescript
const stack = [];
for (const char of s) {
  if (char === '여는문자') {
    stack.push(char);
  } else {
    if (stack.length === 0) return false; // 조기 반환
    stack.pop();
  }
}
return stack.length === 0;
```
